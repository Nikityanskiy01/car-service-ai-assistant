# Памятка: как запустить AI Fox Motors локально

Кратко: один раз ставите зависимости backend, настраиваете `.env`, применяете миграции Prisma, заливаете seed и запускаете `npm run dev`. Фронтенд раздаётся тем же процессом Express — отдельный сервер для HTML не нужен.

---

## 1. Что должно быть установлено

| Компонент | Версия / примечание |
|-----------|---------------------|
| **Node.js** | 20 LTS или новее (`node -v`) |
| **npm** | идёт вместе с Node (`npm -v`) |
| **Git** | если клонируете репозиторий (опционально) |

**База данных:** Prisma настроен на **PostgreSQL**; строка подключения задаётся через `DATABASE_URL` в `backend/.env`. Для локального запуска удобно поднять PostgreSQL из `docker-compose.yml` в корне репозитория (`docker compose up -d`), затем создать БД при необходимости и выполнить миграции (см. ниже).

**LLM:** для работы ИИ-консультаций нужен OpenAI-compatible сервер (например Ollama) **или** режим заглушки без реальной модели.

---

## 2. Получить код

Если репозиторий уже на диске — откройте папку проекта в терминале. Если клонируете:

```bash
git clone <url-репозитория>
cd AI-fox-motors
```

Дальше все команды ниже предполагают, что вы находитесь в **корне репозитория** или явно переходите в `backend/`.

---

## 3. Установить зависимости backend

Из корня репозитория:

```bash
cd backend
npm install
```

Корневой `package.json` в основном для **E2E-тестов Playwright**; для обычной разработки сайта достаточно зависимостей в `backend/`.

---

## 4. Файл окружения `.env`

1. Скопируйте пример:

   **PowerShell (Windows):**

   ```powershell
   cd backend
   Copy-Item .env.example .env
   ```

   **bash:**

   ```bash
   cd backend
   cp .env.example .env
   ```

2. Откройте `backend/.env` и при необходимости измените:

   | Переменная | Назначение |
   |------------|------------|
   | `DATABASE_URL` | По умолчанию в `.env.example`: `postgresql://fox:fox@localhost:5432/foxmotors` (после `docker compose up`) |
   | `PORT` | Порт HTTP-сервера (по умолчанию **3000**) |
   | `JWT_SECRET` | Секрет для JWT; в продакшене замените на длинную случайную строку |
   | `CORS_ORIGIN` | Origin для CORS (часто `http://localhost:3000` или `http://127.0.0.1:3000`) |
   | `LLM_BASE_URL` | Базовый URL OpenAI-compatible API (например Ollama: `http://127.0.0.1:11434/v1`) |
   | `LLM_MODEL` | Имя модели (например `llama3.2`) |
   | `LLM_MOCK` | **`1`** — ответы без реального LLM (удобно для быстрого старта и тестов); **`0`** — реальные запросы к модели |
   | `TELEGRAM_BOT_TOKEN`, `TELEGRAM_MANAGER_CHAT_IDS` | Опционально; без токена уведомления могут оставаться в «демо»-режиме в БД |

---

## 5. Миграции и начальные данные

В каталоге `backend/`:

```bash
npx prisma migrate deploy
npm run db:seed
```

- **`migrate deploy`** применяет уже существующие миграции из `prisma/migrations/`.
- **`db:seed`** создаёт тестовых пользователей и прочие начальные данные.

Если при разработке вы меняете схему и создаёте новые миграции, обычно используют `npx prisma migrate dev` (это уже сценарий для разработчиков схемы БД).

**Просмотр данных в GUI:** `npm run db:studio` (Prisma Studio).

---

## 6. Запуск в режиме разработки

```bash
cd backend
npm run dev
```

Скрипт `dev` запускает **nodemon** и перезагружает сервер при изменении файлов. API и статика фронтенда обслуживаются одним процессом: статика берётся из папки `frontend/` относительно корня репозитория.

Откройте в браузере:

**http://127.0.0.1:3000/**

(если не меняли `PORT`).

Продакшен-старт без автоперезагрузки: `npm run start`.

---

## 7. LLM: два варианта

### Вариант A — без установки модели (быстро)

В `backend/.env`:

```env
LLM_MOCK=1
```

Консультации будут работать через заглушку, без Ollama.

### Вариант B — локальная модель (Ollama, Windows)

1. Установите [Ollama](https://ollama.com/) и запустите сервис.
2. Скачайте модель, например:

   ```bash
   ollama pull llama3.2
   ```

3. В `.env`:

   ```env
   LLM_MOCK=0
   LLM_BASE_URL=http://127.0.0.1:11434/v1
   LLM_MODEL=llama3.2
   ```

4. Проверка из `backend/`:

   ```bash
   npm run llm:check
   ```

Если проверка проходит, запросы к ИИ пойдут в локальную модель.

## 9. Типичные проблемы

| Симптом | Что проверить |
|---------|----------------|
| Порт занят | Смените `PORT` в `.env` или освободите порт (часто 3000). |
| Ошибки Prisma / «таблица не найдена» | Выполните `npx prisma migrate deploy` из `backend/`. |
| Страница не открывается | Убедитесь, что сервер запущен и вы открываете тот же хост/порт (`127.0.0.1` vs `localhost` иногда влияет на CORS — сверьте `CORS_ORIGIN`). |
| ИИ не отвечает | `LLM_MOCK=1` для проверки без LLM; при `LLM_MOCK=0` — запущен ли Ollama и совпадают ли `LLM_BASE_URL` / `LLM_MODEL`. |
| PostgreSQL недоступен | Запущен ли контейнер (`docker compose ps`), верны ли хост/порт/логин в `DATABASE_URL`, создана ли БД. |

---

## 10. Дополнительно

- **Юнит/интеграционные тесты backend:** `cd backend && npm test` (при необходимости задайте `TEST_DATABASE_URL` или используйте тот же `DATABASE_URL`, см. `README.md`).
- **E2E (Playwright):** из корня — `npm install`, затем `npx playwright install` и `npm run test:e2e` (нужен запущенный backend; часто `LLM_MOCK=1`).
- Контракт API: `specs/001-ai-consultation-platform/contracts/openapi.yaml`.

Актуальная схема БД и переменные окружения — в `backend/prisma/schema.prisma` и `README.md`.
