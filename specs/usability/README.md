## Юзабилити-тесты (быстрый прогон)

Цель: быстро находить UX-проблемы и улучшать конверсию в консультацию/заявку.

### Ключевые сценарии (за 10–15 минут)

1) **Гость → консультация → заявка**
- Открыть главную
- Перейти в ИИ-консультацию
- Начать новую сессию
- Заполнить авто + симптомы + условия
- Создать заявку **без регистрации** (имя + телефон)

Ожидаемое:
- Понятно что вводить, нет “угадываний”
- Нет лишних блоков после завершения
- Контактные данные запрашиваются один раз

2) **Гость → регистрация → перенос заявки**
- Создать гостевую заявку
- Зарегистрироваться/войти
- Открыть ЛК клиента

Ожидаемое:
- Заявка появляется в “Заявки”
- Консультация привязана к аккаунту

3) **Клиент → отслеживание и переписка**
- Войти как клиент
- Открыть “Заявки”
- Прочитать переписку, отправить сообщение
- Открыть “Записи”, создать запись

4) **Менеджер → обработка**
- Войти как менеджер
- Открыть заявку, сменить статус
- Ответить клиенту

### Эвристики (чек-лист)
- **Ясность**: на экране один главный CTA
- **Обратная связь**: после действий есть подтверждение/ошибка
- **Ошибки**: понятное сообщение, как исправить
- **Согласованность**: одинаковые кнопки/шрифты/карточки по всему сайту
- **Мобильная версия**: меню, формы, таблицы не “ломаются”

### Найдено в walkthrough и исправлено (2026-03-25)

- **Гость в консультации**: конфликт CTA (“войдите чтобы оформить заявку” при наличии кнопки гостевой заявки) → заменено на явный выбор:
  - “Создать заявку (гость)” или “Войти/Регистрация” для отслеживания.
- **После гостевой заявки**: редирект в ЛК без авторизации → заменён на редирект на вход с `next` на вкладку “Заявки”.
- **Привязка guest→account (claim)**: добавлена понятная обратная связь в ЛК после успешной привязки; очищаются гостевые маркеры в `sessionStorage`.
- **Кабинеты**: формы переписки прячутся до выбора заявки; добавлены пустые состояния (чтобы не было “пустых белых блоков”).
- **Навигация**: на страницах кабинетов подсвечивается пункт “Кабинет” в шапке (чтобы было понятно, где вы находитесь).

### Что проверить на мобилке (быстрый чек)

- В шапке: кнопка “Меню” открывает/закрывает навигацию, ссылки кликабельны, нет налезания кнопок.
- Таблицы в кабинетах: горизонтальный скролл работает, выбранная строка видна.
- Модальные окна: прокрутка внутри, закрытие Esc/кнопкой, фокус не “теряется”.
