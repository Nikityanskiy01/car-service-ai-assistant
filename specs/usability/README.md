## Юзабилити-тесты (быстрый прогон)

Цель: быстро находить UX-проблемы и улучшать конверсию в консультацию/заявку.

### Ключевые сценарии (за 10–15 минут)

1) **Гость → консультация → заявка**
- Открыть главную
- Перейти в ИИ-консультацию
- Начать новую сессию
- Заполнить авто + симптомы + условия
- Создать заявку **без регистрации** (имя + телефон)

Ожидаемое:
- Понятно что вводить, нет “угадываний”
- Нет лишних блоков после завершения
- Контактные данные запрашиваются один раз

2) **Гость → регистрация → перенос заявки**
- Создать гостевую заявку
- Зарегистрироваться/войти
- Открыть ЛК клиента

Ожидаемое:
- Заявка появляется в “Заявки”
- Консультация привязана к аккаунту

3) **Клиент → отслеживание и переписка**
- Войти как клиент
- Открыть “Заявки”
- Прочитать переписку, отправить сообщение
- Открыть “Записи”, создать запись

4) **Менеджер → обработка**
- Войти как менеджер
- Открыть заявку, сменить статус
- Ответить клиенту

### Эвристики (чек-лист)
- **Ясность**: на экране один главный CTA
- **Обратная связь**: после действий есть подтверждение/ошибка
- **Ошибки**: понятное сообщение, как исправить
- **Согласованность**: одинаковые кнопки/шрифты/карточки по всему сайту
- **Мобильная версия**: меню, формы, таблицы не “ломаются”
